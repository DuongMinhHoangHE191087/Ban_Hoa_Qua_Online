📌 Important Notes

Mechanism Description
Multi-Shop Splitting If the cart contains products from multiple shops, the system automatically creates 1 Parent Order + N Child Orders (one per shop). Each child order is processed independently by the respective Shop Owner.
Inventory Reserve When an order is placed, stock is temporarily reserved within the transaction. If the order is cancelled, the reserved stock is restored.
Concurrency Lock The system uses UserLockManager (ReentrantLock per user) to prevent double-submit, ensuring only one place-order request is processed per user at any given time.
SePay Webhook Dedup The sepay_webhook_dedup table stores sepayTxId to ensure each bank transaction is processed exactly once (idempotent).
Manual Fallback If the SePay Webhook is delayed or fails, the customer can click "I have already paid" → the order transitions to processing status for Admin to manually verify.
COD Eligibility COD is only available for orders under 2,000,000 VND and for accounts without excessive failed deliveries (OrderService.isCodEligible()).
