Tài liệu này giải thích chi tiết từng lớp đối tượng (Class) tham gia vào quy trình **Restock Management** (Nhập hàng). Dưới đây là mô tả chi tiết từng dòng code cốt lõi, lý do lựa chọn cú pháp và lợi ích bảo mật/vận hành thu được.

1. Tổng Quan Lớp Đối Tượng Liên Quan

Tên Lớp (Class Name) Tầng (Layer) Trách Nhiệm Chính (Core Responsibility)
InventoryLog.java Entity Model Ánh xạ cấu trúc bảng inventory_logs trong CSDL và chứa các thuộc tính bổ trợ hiển thị.

2. Lớp InventoryLog.java (Entity Model)

Mã nguồn định nghĩa các thuộc tính ánh xạ cơ bản và thuộc tính hiển thị liên kết:

public class InventoryLog {
    private int logId;
    private int variantId;
    private int changedBy;
    private String changeType;
    private int quantityDelta;
    private int quantityAfter;
    private String note;
    private LocalDateTime changedAt;

    // Helper fields for UI display (not mapped in database tables)
    private String productName;
    private String variantLabel;
}
Đoạn Mã / Thuộc tính Ý nghĩa & Lý do thực hiện Lợi ích thu được
private int variantId; Ánh xạ khóa ngoại variant_id. Nhập kho sẽ nhắm thẳng tới biến thể sản phẩm cụ thể. Cho phép kiểm soát tồn kho ở mức độ chi tiết nhất (ví dụ: quả táo đỏ loại 1kg, 2kg có tồn kho riêng).
private int quantityDelta; Hiển thị số lượng thay đổi (+ khi nhập kho, - khi bán hàng/hủy hàng). Cho phép ghi nhận cả nghiệp vụ tăng lẫn giảm tồn kho một cách thống nhất trên một bảng dữ liệu.
private String productName;
private String variantLabel;		 Các thuộc tính bổ trợ (helper fields) không ghi vào CSDL nhưng được SELECT kèm qua câu lệnh JOIN. Tránh vấn đề N+1 query: JSP chỉ cần đọc trực tiếp thông tin tên sản phẩm từ đối tượng Log, không cần gọi query lẻ sang bảng Product/Variant để lấy tên hiển thị.

3. Lớp InventoryDAO.java (Data Access Object)

Trực tiếp tương tác với dữ liệu. Dưới đây là giải thích chi tiết các phương thức nghiệp vụ:

Hàm openConnection()

public Connection openConnection() throws SQLException {

}
Dòng mã Ý nghĩa & Lý do thực hiện Lợi ích thu được
public Connection openConnection() Expose phương thức getConnection() kế thừa từ lớp cha BaseDAO (vốn là protected) thành public. Cho phép lớp Service chủ động lấy Connection để tự quản lý vòng đời Transaction (Commit/Rollback), trong khi Servlet hay các tầng khác vẫn không can thiệp trực tiếp vào kết nối gốc.

Hàm save(Connection conn, InventoryLog log)

String sql = "INSERT INTO inventory_logs (variant_id, changed_by, change_type, quantity_delta, quantity_after, note, changed_at) VALUES (?, ?, ?, ?, ?, ?, GETDATE())";
try (PreparedStatement ps = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) {
    ps.setInt(1, log.getVariantId());
    ps.setInt(2, log.getChangedBy());
    ps.setString(3, log.getChangeType());
    ps.setInt(4, log.getQuantityDelta());
    ps.setInt(5, log.getQuantityAfter());
    ps.setString(6, log.getNote());
    ps.executeUpdate();
}
Dòng mã Ý nghĩa & Lý do thực hiện Lợi ích thu được
Connection conn (tham số truyền vào) Nhận đối tượng Connection đang chạy dở của Transaction được mở ra từ tầng Service. Đảm bảo câu lệnh SQL INSERT này chạy chung một kết nối vật lý với câu lệnh UPDATE tồn kho, phục vụ mục đích commit/rollback đồng bộ.
Statement.RETURN_GENERATED_KEYS Yêu cầu CSDL trả về các cột khóa tự tăng (Identity) vừa mới được sinh ra. Giúp lấy được log_id tự sinh ngay sau khi insert, phục vụ mục đích hiển thị ngay lập tức lên UI mà không cần SELECT lại.

4. Lớp InventoryService.java (Service Layer - Transaction Orchestrator)

Tầng quản lý nghiệp vụ và thực thi giao dịch. Đảm bảo tính nguyên tử (Atomicity):

try (Connection conn = inventoryDAO.openConnection()) {
    conn.setAutoCommit(false);
    try {
        int currentStock = productVariantDAO.getStockQuantity(conn, variantId);
        int stockAfter = currentStock + quantity;

        InventoryLog log = new InventoryLog();
        log.setVariantId(variantId);
        log.setChangedBy(userId);
        log.setChangeType("MANUAL_ADJUST");
        log.setQuantityDelta(quantity);
        log.setQuantityAfter(stockAfter);
        log.setNote(note != null && !note.trim().isEmpty() ? note.trim() : "Nhập kho");
        log.setChangedAt(changedAt.atTime(LocalTime.now()));

        inventoryDAO.save(conn, log);
        productVariantDAO.updateStock(conn, variantId, quantity);
    
        conn.commit();
    } catch (Exception e) {
        conn.rollback();
        throw e;
    } finally {
        conn.setAutoCommit(true);
    }
}
Dòng mã Ý nghĩa & Lý do thực hiện Lợi ích thu được
try (Connection conn = ...) Sử dụng cú pháp try-with-resources để khai báo và khởi tạo kết nối Database. Chống rò rỉ kết nối (Connection Leak): Java sẽ tự động đóng kết nối (gọi conn.close()) trong mọi trường hợp, bảo vệ tài nguyên Server.
conn.setAutoCommit(false); Tắt chế độ tự động xác nhận câu lệnh của JDBC. Đánh dấu điểm bắt đầu của một Database Transaction. Mọi câu lệnh SQL chạy sau dòng này sẽ chỉ nằm ở trạng thái chờ xác nhận.
conn.commit(); Đồng ý lưu tất cả thay đổi (cả phiếu Log và số lượng cập nhật) vào ổ đĩa. Đảm bảo tính nhất quán (Consistency) của dữ liệu: cả hai tác vụ cùng hoàn thành.
conn.rollback(); Hủy bỏ toàn bộ các câu lệnh đã chạy từ lúc đặt setAutoCommit(false). Hoàn trả cơ sở dữ liệu về trạng thái an toàn trước khi xảy ra lỗi. Không để xảy ra tình trạng mất mát dữ liệu hoặc log mồ côi.

5. Lớp InventoryServlet.java (Controller / Security Guard)

Nơi tiếp nhận Request, thực thi các biện pháp phòng vệ và điều hướng dữ liệu:

Luồng xử lý GET (doGet)

User currentUser = SessionUtil.getCurrentUser(session);
if (currentUser == null || !AppConfig.ROLE_SHOP_OWNER.equals(currentUser.getRole())) {
    resp.sendRedirect(req.getContextPath() + "/auth/login");
    return;
}
Dòng mã Ý nghĩa & Lý do thực hiện Lợi ích thu được
!AppConfig.ROLE_SHOP_OWNER.equals(...) Phân quyền ở mức Servlet. Kiểm tra vai trò của tài khoản trong session. Bảo mật tài nguyên hệ thống: Ngăn chặn khách hàng (CUSTOMER) hay tài xế (DELIVERY) cố tình đoán URL nhập kho để tự tiện tăng số lượng hàng hóa.

Luồng xử lý POST (doPost - Chống giả mạo)

ProductVariant pv = productVariantDAO.findById(variantId);
List<Product> products = productDAO.findById(pv.getProductId());
if (products.isEmpty() || products.get(0).getOwnerId() != currentUser.getUserId()) {
    SessionUtil.flashError(session, "Bạn không có quyền nhập kho cho sản phẩm này.");
    resp.sendRedirect(req.getContextPath() + "/shop/inventory");
    return;
}
Dòng mã Ý nghĩa & Lý do thực hiện Lợi ích thu được
products.get(0).getOwnerId() != currentUser.getUserId() Security Ownership Check (IDOR Prevention): Truy vết từ Biến thể ➔ Sản phẩm ➔ Lấy ID chủ shop của sản phẩm đó và so sánh với ID người dùng hiện tại. Ngăn chặn việc tin tặc dùng Inspect Element để đổi giá trị variantId trong thẻ <select> thành ID sản phẩm của shop khác để thực hiện nhập hàng khống.
resp.sendRedirect(...) Sau khi xử lý thành công hoặc thất bại, gửi tín hiệu chuyển hướng về trình duyệt. PRG (Post-Redirect-Get) Pattern: Giải quyết triệt để lỗi người dùng nhấn F5/Refresh trang làm gửi lại Form nhập kho nhiều lần (dẫn tới bị cộng dồn số lượng ảo liên tiếp).
