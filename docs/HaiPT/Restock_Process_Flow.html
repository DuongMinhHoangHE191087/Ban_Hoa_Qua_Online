🔄 Restock Process Flow

Ban Hoa Qua Online — Full Inventory Restock Lifecycle

System Overview 3 Flows

🔁 Product Lifecycle

Products automatically transition from ACTIVE → OUT_OF_SEASON when their harvest date + shelf-life expires.

  • Triggered by autoDeactivateExpiredProducts()
  • Runs on every product listing query
  • SQL: harvest_date + shelf_life_days ≤ today
  • Class: ProductDAO → ProductService

🛒 Customer Restock Request

Logged-in customers viewing an OUT_OF_SEASON product can request the Shop Owner to restock the new harvest.

  • Rate-limited: 1 request / customer / product / day
  • Creates a notifications row for Shop Owner
  • INACTIVE products are blocked entirely
  • Class: ProductDetailServlet POST

🏪 Shop Owner Restock

Shop Owner fills the restock form at /shop/inventory, which executes a DB transaction to update stock and reactivate the product.

  • Validates: quantity > 0, date ≤ today, ownership
  • DB Transaction: log + stock + harvest_date + status
  • Product status: → ACTIVE
  • PRG pattern after POST
Status Transition Map Product Statuses
ACTIVE → OUT_OF_SEASON → ACTIVE
INACTIVE (manual deactivation by owner)
DELETED (soft delete — not accessible)

Restock is the mechanism that transitions a product from OUT_OF_SEASON back to ACTIVE, by updating harvest_date and status inside a single DB transaction.

🔗 Atomic Transaction — All-or-Nothing
1

Error Handling Summary Exception Paths
Error Paths → Flash + Redirect
  • Missing required fields → "Vui lòng nhập đầy đủ các trường bắt buộc"
  • Invalid number format → "Mã sản phẩm hoặc số lượng không hợp lệ"
  • Invalid date format → "Định dạng ngày nhập kho không hợp lệ"
  • Quantity ≤ 0 → "Số lượng nhập kho phải lớn hơn 0"
  • Future date → "Ngày nhập kho không được lớn hơn ngày hiện tại"
  • Variant not found → "Biến thể sản phẩm không tồn tại"
  • Not owner → "Bạn không có quyền nhập kho cho sản phẩm này"
  • SQLException → "Lỗi cơ sở dữ liệu: ..."
  • Unknown exception → "Đã xảy ra lỗi không xác định"
All errors use SessionUtil.flashError() + sendRedirect("/shop/inventory") (PRG)
