🚀 Báo Cáo Tiến Độ Cập Nhật Code (Phase 1 & 2)

Ngày cập nhật: 31/05/2026

Báo cáo này tổng hợp toàn bộ các tệp tin đã được tạo mới, chỉnh sửa và các lỗi đã được khắc phục trong suốt quá trình hoàn thiện Phase 1 (Quản lý người dùng) và Phase 2 (Phê duyệt cửa hàng & Kiểm duyệt đánh giá) dành cho Admin.

1. Tính Năng Mới (New Features)

  • NEW AdminUserServlet.java & AdminUserAPI.java
    Tạo mới Controller xử lý hiển thị danh sách người dùng, tìm kiếm, phân trang và API gọi AJAX để Khóa/Mở khóa tài khoản không cần load lại trang.
  • NEW ShopApprovalServlet.java & ShopApprovalAPI.java
    Tạo mới Controller lấy danh sách 3 cửa hàng từ Database để Admin phê duyệt hoặc từ chối, kèm API xử lý AJAX.
  • NEW AdminReviewServlet.java & AdminReviewAPI.java
    Tạo mới Controller quản lý danh sách đánh giá của khách hàng, cho phép Admin Ẩn/Hiện các bình luận vi phạm.
  • NEW ShopService.java & UserService.java
    Tạo mới tầng Service để tách biệt Business Logic (xử lý nghiệp vụ) ra khỏi DAO và Servlet, đảm bảo đúng kiến trúc MVC.
  • NEW Các file JSP giao diện Admin
    - user-management.jsp (Quản lý User)
    - shop-approvals.jsp (Phê duyệt Shop)
    - review-management.jsp (Kiểm duyệt Đánh giá)
    - coming-soon.jsp (Trang chờ cho các tính năng Phase 4 chưa ra mắt).

3. Các Lỗi Đã Khắc Phục (Bug Fixes)

  • FIX Lỗi JDK 26 (Không compile được code)
    Nguyên nhân: project.properties của NetBeans đang set javac.source=26 (bản Java không tồn tại hoặc chưa hỗ trợ), khiến code mới không được nạp vào Tomcat, gây ra hiện tượng "trống rỗng".
    Giải quyết: Hạ cấu hình xuống chuẩn javac.source=17 và compile lại thành công.
  • FIX Lỗi 500 "Invalid column name product_name"
    Nguyên nhân: Trong Database bảng Products dùng cột name, nhưng trong ReviewDAO lại truy vấn nhầm thành p.product_name gây lỗi SQL ném ra 500.
    Giải quyết: Sửa lại thành p.name AS product_name.
  • FIX Lỗi 404 Nút Đối soát thanh toán
    Nguyên nhân: Thiếu chữ "s". Sidebar trỏ tới /admin/settlements nhưng Servlet lại khai báo /admin/settlement.
    Giải quyết: Thêm chữ "s" vào @WebServlet.
  • FIX Lỗi 500 Các tính năng Phase 4
    Nguyên nhân: Code mặc định ném ra Exception vì chưa làm.
    Giải quyết: Thay thế bằng giao diện coming-soon.jsp thân thiện hơn với người dùng.
