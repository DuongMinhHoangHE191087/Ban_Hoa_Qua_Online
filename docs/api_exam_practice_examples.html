2. Bài 1 — API lấy top 5 sản phẩm theo doanh thu

2.1. Chốt nghiệp vụ trước khi viết SQL

  • Chỉ tính đơn DELIVERED, vì đây là doanh thu đã hoàn tất.
  • Chỉ tính order_type = 'CHILD', tránh tính trùng parent và child trong đơn nhiều shop.
  • Doanh thu sản phẩm = SUM(order_items.subtotal); số lượng bán = SUM(order_items.quantity).
  • Không lọc products.status = ACTIVE khi xem lịch sử báo cáo, vì sản phẩm đã ngừng bán vẫn có doanh thu cũ.
  • Sắp xếp phụ bằng product_id ASC để hai sản phẩm cùng doanh thu luôn có thứ tự ổn định.
Câu hỏi hay bị bắt lỗi: Nếu dùng SUM(orders.final_amount) theo product thì một đơn có nhiều sản phẩm sẽ làm doanh thu bị lặp. Nếu muốn “net revenue sau coupon”, phải thiết kế quy tắc phân bổ discount cho từng order item; không được tự ý cộng final_amount vào từng sản phẩm.

2.4. DAO mới chứa SQL

Tạo file minh họa mới: src/java/dao/report/ProductRevenueReportDAO.java. Không đặt đoạn SQL này vào Servlet.

package dao.report;

import dao.system.BaseDAO;
import model.dto.report.TopRevenueProductDTO;

import java.math.BigDecimal;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Timestamp;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

public class ProductRevenueReportDAO extends BaseDAO {

    public List<TopRevenueProductDTO> findTopRevenue(
            Integer ownerId, LocalDate from, LocalDate to, int limit)
            throws SQLException {

        StringBuilder sql = new StringBuilder(
            "SELECT p.product_id, p.name, " +
            "SUM(oi.subtotal) AS revenue, " +
            "SUM(oi.quantity) AS sold_quantity " +
            "FROM orders o " +
            "JOIN order_items oi ON oi.order_id = o.order_id " +
            "JOIN product_variants pv ON pv.variant_id = oi.variant_id " +
            "JOIN products p ON p.product_id = pv.product_id " +
            "WHERE o.status = 'DELIVERED' " +
            "AND o.order_type = 'CHILD' ");

        List<Object> params = new ArrayList<>();

        if (ownerId != null) {
            sql.append("AND o.owner_id = ? ");
            params.add(ownerId);
        }
        if (from != null) {
            sql.append("AND o.created_at >= ? ");
            params.add(Timestamp.valueOf(from.atStartOfDay()));
        }
        if (to != null) {
            sql.append("AND o.created_at < ? ");
            params.add(Timestamp.valueOf(to.plusDays(1).atStartOfDay()));
        }

        sql.append(
            "GROUP BY p.product_id, p.name " +
            "ORDER BY SUM(oi.subtotal) DESC, p.product_id ASC " +
            "OFFSET 0 ROWS FETCH NEXT ? ROWS ONLY");
        params.add(limit);

        List<TopRevenueProductDTO> result = new ArrayList<>();
        try (Connection conn = getConnection();
             PreparedStatement ps = conn.prepareStatement(sql.toString())) {

            for (int i = 0; i < params.size(); i++) {
                ps.setObject(i + 1, params.get(i));
            }
            try (ResultSet rs = ps.executeQuery()) {
                while (rs.next()) {
                    result.add(new TopRevenueProductDTO(
                        rs.getInt("product_id"),
                        rs.getString("name"),
                        rs.getBigDecimal("revenue"),
                        rs.getInt("sold_quantity"),
                        "VND"));
                }
            }
        }
        return result;
    }
}

package service.report;

import dao.report.ProductRevenueReportDAO;
import model.dto.report.TopRevenueProductDTO;

import java.sql.SQLException;
import java.time.LocalDate;
import java.util.List;

public class ProductRevenueReportService {
    private final ProductRevenueReportDAO reportDAO =
            new ProductRevenueReportDAO();

    public List<TopRevenueProductDTO> getTopRevenue(
            Integer ownerId, LocalDate from, LocalDate to, int limit)
            throws SQLException {

        if (limit < 1 || limit > 100) {
            throw new IllegalArgumentException("limit phải từ 1 đến 100.");
        }
        if (from != null && to != null && from.isAfter(to)) {
            throw new IllegalArgumentException("Khoảng ngày không hợp lệ.");
        }
        return reportDAO.findTopRevenue(ownerId, from, to, limit);
    }
}

2.6. Servlet API chỉ làm HTTP và quyền

Tạo file minh họa mới: src/java/servlet/api/report/TopRevenueProductApiServlet.java. Đây là API GET đọc dữ liệu nên không cần CSRF, nhưng vẫn cần authentication và authorization.

@WebServlet("/api/v1/admin/reports/products/top-revenue")
public class TopRevenueProductApiServlet extends HttpServlet {
    private final ProductRevenueReportService service =
            new ProductRevenueReportService();

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws IOException {
        resp.setContentType("application/json;charset=UTF-8");

        User user = SessionUtil.getCurrentUser(req.getSession(false));
        if (user == null) {
            resp.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            JsonUtil.writeJson(resp, ApiResponse.fail(401,
                    "Vui lòng đăng nhập."));
            return;
        }
        if (!AppConfig.ROLE_ADMIN.equals(user.getRole())) {
            resp.setStatus(HttpServletResponse.SC_FORBIDDEN);
            JsonUtil.writeJson(resp, ApiResponse.fail(403,
                    "Bạn không có quyền xem báo cáo."));
            return;
        }

        try {
            int limit = parseLimit(req.getParameter("limit"), 5);
            LocalDate from = parseDate(req.getParameter("from"));
            LocalDate to = parseDate(req.getParameter("to"));

            List<TopRevenueProductDTO> data =
                    service.getTopRevenue(null, from, to, limit);

            resp.setStatus(HttpServletResponse.SC_OK);
            JsonUtil.writeJson(resp, ApiResponse.ok(data));
        } catch (IllegalArgumentException e) {
            resp.setStatus(HttpServletResponse.SC_BAD_REQUEST);
            JsonUtil.writeJson(resp, ApiResponse.fail(400,
                    "Tham số báo cáo không hợp lệ."));
        } catch (SQLException e) {
            resp.setStatus(HttpServletResponse.SC_INTERNAL_SERVER_ERROR);
            JsonUtil.writeJson(resp, ApiResponse.fail(500,
                    "Không thể tải báo cáo."));
        }
    }

    private int parseLimit(String raw, int defaultValue) {
        if (raw == null || raw.isBlank()) return defaultValue;
        return Integer.parseInt(raw);
    }

    private LocalDate parseDate(String raw) {
        return raw == null || raw.isBlank() ? null : LocalDate.parse(raw);
    }
}
Ghi chú tích hợp: @WebServlet là route mới. Trong project hiện tại, /api/* đi qua AuthFilter nhưng RoleFilter không tự bảo vệ mọi route API; vì vậy Servlet hoặc Service mới phải tự kiểm tra ADMIN. Không lấy ownerId từ query nếu endpoint dành cho shop owner; phải derive từ session.

3.4. Service tính giá và validate

Đây là nơi đặt business rule, không đặt trong JSP và không đặt trong SQL.

public class ProductCreationService {
    private static final BigDecimal ORGANIC_MULTIPLIER =
            new BigDecimal("1.05");

    private final ProductCreationDAO creationDAO =
            new ProductCreationDAO();

    public CreatedProductResult create(
            User actor, CreateProductRequest request)
            throws SQLException {

        if (actor == null ||
                !AppConfig.ROLE_SHOP_OWNER.equals(actor.getRole())) {
            throw new IllegalArgumentException("Chỉ shop owner được tạo sản phẩm.");
        }
        validate(request);

        Product product = new Product();
        product.setOwnerId(actor.getUserId());
        product.setCategoryId(request.categoryId());
        product.setName(request.name().trim());
        product.setDescription(request.description());
        product.setIsOrganic(request.isOrganic());
        product.setStatus("ACTIVE");
        product.setApprovalStatus("PENDING");

        List<ProductVariant> variants = new ArrayList<>();
        for (CreateVariantRequest input : request.variants()) {
            ProductVariant variant = new ProductVariant();
            variant.setSku(input.sku().trim());
            variant.setVariantLabel(input.label().trim());

            BigDecimal finalPrice = input.basePrice();
            if (request.isOrganic()) {
                finalPrice = finalPrice
                        .multiply(ORGANIC_MULTIPLIER)
                        .setScale(2, RoundingMode.HALF_UP);
            }
            variant.setPrice(finalPrice);
            variant.setStockQuantity(input.stockQuantity());
            variant.setIsActive(true);
            variants.add(variant);
        }

        return creationDAO.insertProductWithVariants(product, variants);
    }

    private void validate(CreateProductRequest request) {
        if (request == null || request.name() == null
                || request.name().isBlank()) {
            throw new IllegalArgumentException("Tên sản phẩm bắt buộc.");
        }
        if (request.categoryId() == null || request.categoryId() <= 0) {
            throw new IllegalArgumentException("Danh mục không hợp lệ.");
        }
        if (request.variants() == null || request.variants().isEmpty()) {
            throw new IllegalArgumentException("Phải có ít nhất một biến thể.");
        }
        if (request.isOrganic()
                && request.verificationDocumentId() == null) {
            throw new IllegalArgumentException(
                    "Sản phẩm hữu cơ phải có chứng từ xác minh.");
        }
        for (CreateVariantRequest variant : request.variants()) {
            if (variant.basePrice() == null
                    || variant.basePrice().compareTo(BigDecimal.ZERO) <= 0) {
                throw new IllegalArgumentException("Giá cơ sở phải lớn hơn 0.");
            }
            if (variant.stockQuantity() < 0) {
                throw new IllegalArgumentException("Tồn kho không được âm.");
            }
        }
    }
}

3.5. DAO transaction mới

Vì tạo product và nhiều variants phải cùng thành công, dùng một transaction. SQL nằm trong DAO mới ProductCreationDAO.java.

public class ProductCreationDAO extends BaseDAO {
    public CreatedProductResult insertProductWithVariants(
            Product product, List<ProductVariant> variants)
            throws SQLException {

        return withTransaction(conn -> {
            int productId = insertProduct(conn, product);

            for (ProductVariant variant : variants) {
                insertVariant(conn, productId, variant);
            }

            return new CreatedProductResult(productId, variants);
        });
    }

    private int insertProduct(Connection conn, Product p)
            throws SQLException {
        String sql =
            "INSERT INTO products " +
            "(owner_id, category_id, name, description, status, " +
            "is_organic, approval_status, created_at, updated_at) " +
            "VALUES (?, ?, ?, ?, ?, ?, ?, GETDATE(), GETDATE())";

        try (PreparedStatement ps = conn.prepareStatement(
                sql, Statement.RETURN_GENERATED_KEYS)) {
            ps.setInt(1, p.getOwnerId());
            ps.setInt(2, p.getCategoryId());
            ps.setString(3, p.getName());
            ps.setString(4, p.getDescription());
            ps.setString(5, p.getStatus());
            ps.setBoolean(6, p.getIsOrganic());
            ps.setString(7, p.getApprovalStatus());
            ps.executeUpdate();

            try (ResultSet rs = ps.getGeneratedKeys()) {
                if (!rs.next()) throw new SQLException(
                        "Không lấy được product_id.");
                return rs.getInt(1);
            }
        }
    }

    private void insertVariant(Connection conn, int productId,
                               ProductVariant v) throws SQLException {
        String sql =
            "INSERT INTO product_variants " +
            "(product_id, sku, variant_label, price, stock_quantity, " +
            "is_active) VALUES (?, ?, ?, ?, ?, ?)";

        try (PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setInt(1, productId);
            ps.setString(2, v.getSku());
            ps.setString(3, v.getVariantLabel());
            ps.setBigDecimal(4, v.getPrice());
            ps.setInt(5, v.getStockQuantity());
            ps.setBoolean(6, v.getIsActive());
            ps.executeUpdate();
        }
    }
}

@WebServlet("/api/v1/products")
public class ProductCreateApiServlet extends HttpServlet {
    private final ProductCreationService service =
            new ProductCreationService();

    @Override
    protected void doPost(HttpServletRequest req,
                          HttpServletResponse resp)
            throws IOException {
        resp.setContentType("application/json;charset=UTF-8");

        User actor = SessionUtil.getCurrentUser(
                req.getSession(false));
        if (actor == null) {
            resp.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            JsonUtil.writeJson(resp, ApiResponse.fail(401,
                    "Vui lòng đăng nhập."));
            return;
        }
        if (!AppConfig.ROLE_SHOP_OWNER.equals(actor.getRole())) {
            resp.setStatus(HttpServletResponse.SC_FORBIDDEN);
            JsonUtil.writeJson(resp, ApiResponse.fail(403,
                    "Bạn không có quyền tạo sản phẩm."));
            return;
        }

        try {
            String body = req.getReader().lines()
                    .collect(Collectors.joining());
            CreateProductRequest request = JsonUtil.fromJson(
                    body, CreateProductRequest.class);

            CreatedProductResult result =
                    service.create(actor, request);

            resp.setHeader("Location",
                    req.getContextPath() + "/api/v1/products/"
                    + result.productId());
            JsonUtil.writeJson(resp, ApiResponse.ok(result));
        } catch (IllegalArgumentException e) {
            resp.setStatus(HttpServletResponse.SC_BAD_REQUEST);
            JsonUtil.writeJson(resp, ApiResponse.fail(400,
                    e.getMessage()));
        } catch (SQLException e) {
            resp.setStatus(HttpServletResponse.SC_INTERNAL_SERVER_ERROR);
            JsonUtil.writeJson(resp, ApiResponse.fail(500,
                    "Không thể tạo sản phẩm."));
        }
    }
}

5. Checklist kiểm thử

5.1. Top 5 doanh thu

  • ☐ Đơn parent có child không bị tính trùng.
  • ☐ Đơn CONFIRMED/PREPARING chưa DELIVERED không xuất hiện nếu định nghĩa là doanh thu đã thực hiện.
  • ☐ Sản phẩm đã DELETED vẫn có thể xuất hiện trong lịch sử doanh thu.
  • ☐ Hai sản phẩm cùng doanh thu có thứ tự ổn định.
  • ☐ from/to bao phủ đúng cả ngày cuối; from sau to bị 400.
  • ☐ Customer gọi API bị 403; chưa login bị 401; admin nhận danh sách rỗng vẫn 200.

5.2. Tạo sản phẩm hữu cơ

  • ☐ Hữu cơ false: 100000 → 100000.
  • ☐ Hữu cơ true: 100000 → 105000; 270000 → 283500.
  • ☐ Không nhận finalPrice giả từ client.
  • ☐ Giá âm, stock âm, SKU rỗng, không có variant đều bị 400.
  • ☐ Hữu cơ thiếu verification document bị từ chối.
  • ☐ Lỗi insert variant làm rollback product và các variant trước đó.
  • ☐ Customer/admin không phải shop owner bị 403; POST thiếu CSRF bị 403.
  • ☐ Order cũ không bị đổi giá sau khi tạo sản phẩm mới.

6. Cách trả lời khi hội đồng hỏi

Vì sao top doanh thu dùng order_items.subtotal?

Vì subtotal gắn với từng sản phẩm/biến thể. final_amount thuộc toàn đơn, bao gồm phí giao hàng, giảm giá và phí nền tảng; cộng final_amount cho từng sản phẩm sẽ nhân doanh thu.

Vì sao API GET không cần CSRF nhưng POST tạo sản phẩm cần?

GET chỉ đọc dữ liệu và không làm thay đổi trạng thái. POST tạo sản phẩm dùng cookie session nên browser có thể bị ép gửi request; token CSRF chứng minh request đến từ giao diện hợp lệ.

Vì sao tăng 5% ở Service?

Đó là business rule. Đặt ở Service giúp mọi caller đều dùng cùng một quy tắc; client chỉ gửi giá cơ sở và không thể tự gian lận final price.

Vì sao phải transaction khi tạo product?

Một product có nhiều variant. Nếu product insert thành công nhưng variant thứ hai lỗi, transaction rollback để không còn product rỗng hoặc dữ liệu nửa chừng.

Giá hữu cơ có áp dụng hồi tố cho order cũ không?

Không. order_items lưu snapshot unit_price/subtotal tại thời điểm checkout; thay đổi giá sản phẩm hiện tại không được làm sai lịch sử thanh toán.

Tài liệu này là bài ôn tập độc lập. Các class và route mẫu chưa được thêm vào source đang chạy; chỉ tích hợp sau khi đối chiếu schema, filter mapping, SRS và test database.
