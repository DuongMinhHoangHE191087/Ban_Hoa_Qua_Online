Bản Trình Bày & Hướng Dẫn: Xử lý Validation Auth (Logic Input)

Nguyên tắc vàng (Core Rule): Không bao giờ chọc thẳng dữ liệu từ Client vào Database. Trong Java Web (Servlet-DAO), bạn phải gọi Utility Validation để làm sạch đầu vào ở Tầng Servlet & Tầng Service trước khi gọi hàm Insert/Update.

1. Sử Dụng Lớp `ValidationUtil` Đã Có

Dự án của bạn đã có sẵn com.fruitmkt.util.ValidationUtil. Nó cung cấp các Regex chuẩn cho Email, Phone, số nguyên. Ta sẽ import lớp này vào AuthService.java.

2. Tích Hợp Validation Vào Đăng Ký (Register)

Phần này thực hiện ở AuthService.java (Hàm `register`). Phải ném Exception với thông điệp rõ ràng để Servlet chụp lại (Catch) và truyền qua UI báo cho UI bằng Flash Message.


// Import lớp vào đầu file AuthService.java
import com.fruitmkt.util.ValidationUtil;
import com.fruitmkt.util.HashUtil;

public User register(com.fruitmkt.model.entity.User user) throws Exception {
    
    // --- 1. VALIDATION ĐẦU VÀO (Ràng buộc Dữ Liệu) ---
    if (!ValidationUtil.notBlank(user.getFullName())) {
        throw new Exception("Họ tên không được để trống.");
    }
    if (!ValidationUtil.isValidEmail(user.getEmail())) {
        throw new Exception("Định dạng email không hợp lệ.");
    }
    // Ở form đăng ký, user nhập pass qua object User ở cột PasswordHash (do User Entity quy định)
    if (!ValidationUtil.isValidPassword(user.getPasswordHash())) { 
        throw new Exception("Mật khẩu phải từ 8 đến 64 ký tự.");
    }
    // Kiểm tra SĐT nếu có nhập
    if (ValidationUtil.notBlank(user.getPhone()) && !ValidationUtil.isValidPhone(user.getPhone())) {
        throw new Exception("Số điện thoại VN không hợp lệ.");
    }

    // --- 2. VALIDATION NGHIỆP VỤ (Business Rule) ---
    if (userDAO.findByEmail(user.getEmail()) != null) {
        throw new Exception("Email này đã tồn tại trong hệ thống. Hãy đăng nhập!");
    }

    // --- 3. TIẾN HÀNH THÊM MỚI ---
    String hashedPass = HashUtil.hashPassword(user.getPasswordHash());
    int insertedId = userDAO.saveNewCustomer(user.getFullName(), user.getEmail(), hashedPass, user.getPhone());
    
    if (insertedId > 0) {
        return userDAO.findByEmail(user.getEmail()); 
    }
    throw new Exception("Lỗi hệ thống khi tạo tài khoản.");
}

3. Tích Hợp Validation Vào Đăng Nhập (Login)

Trong AuthService.java (Hàm `login`). Chúng ta không cần kiểm tra Regex, chỉ cần kiểm tra xem người dùng có bỏ trống trường hay không, rồi tiếp tục kiểm tra Hash.


public User login(String email, String rawPassword) throws Exception {
    
    // --- 1. VALIDATION CƠ BẢN ---
    if (!ValidationUtil.notBlank(email) || !ValidationUtil.notBlank(rawPassword)) {
        throw new Exception("Vui lòng nhập đầy đủ Email và Mật khẩu.");
    }
    if (!ValidationUtil.isValidEmail(email)) {
        throw new Exception("Định dạng email không hợp lệ.");
    }

    // --- 2. KIỂM TRA TỰ TỒN TẠI ---
    User user = userDAO.findByEmail(email);
    if (user == null) {
        throw new Exception("Tài khoản hoặc mật khẩu không chính xác.");
    }

    // --- 3. KIỂM TRA TRẠNG THÁI KHÓA/HOA T ĐỘNG ---
    if ("INACTIVE".equals(user.getStatus())) {
        throw new Exception("Tài khoản của bạn đã bị vô hiệu hoá. Vui lòng liên hệ Admin.");
    }

    // --- 4. KIỂM TRA MẬT KHẨU ---
    // (Lưu ý thay HashMap bằng BCrypt hoặc tool Hash tương ứng của bạn)
    if (!HashUtil.verifyPassword(rawPassword, user.getPasswordHash())) {
        // Cập nhật số lần nhầm lẫn +1 ở Database để phòng Brute-force
        userDAO.incrementFailedLogin(user.getUserId()); 
        throw new Exception("Tài khoản hoặc mật khẩu không chính xác.");
    }

    // Phục hồi lại số lần sai mật khẩu về 0
    userDAO.resetFailedLogin(user.getUserId());
    
    return user;
}

4. Hướng dẫn Code tầng UI & Servlet (Luồng Bọc Ngoài)

Phần này trả lời cho câu hỏi: Lấy ngoại lệ (Exception) hiển thị lên màn hình ra sao?

  • Servlet: Nhận tham số, gán vào Đối tượng -> Gom hàm try/catch bọc lại.
  • Catch(Exception e): Lấy e.getMessage(), dán vào SessionUtil.setFlashMessage của hệ thống rồi response.sendRedirect ngay lập tức về trang Register.
  • JSP: Code JSP đọc Flash Message và hiện ra Toast thông báo góc màn hình đỏ rực.
