Hướng Dẫn Base: Xây dựng Core Login & Đăng ký bằng Email để chuẩn bị cho Google OAuth

Như yêu cầu, chúng ta định danh người dùng duy nhất dựa qua Email. Ở bước này, chúng ta tập trung tổ chức tải Jackson và xây dựng các hàm ở DAO/Service để tạo và tìm người dùng trước.

1. Thư viện Jackson (Cần người dùng thao tác)

Để parse chuỗi JSON từ Google API (lấy email, tên), bạn cần cài Jackson. Hãy tự làm theo bước sau:

  1. Vào Maven Repository tìm: jackson-databind, jackson-core, jackson-annotations.
  2. Tải các file .jar mới nhất (ví dụ bản 2.15.x).
  3. Copy 3 file .jar này dán vào thư mục thư viện của dự án: web/WEB-INF/lib/ (hoặc ném vào Libraries của NetBeans).

2. Kế hoạch ở tầng Database (DAO)

Ta cần hai thao tác SQL chính cho quá trình Login / OAuth: Tìm user theo email, và lưu user mới.

2.1 Bổ sung hàm findByEmail trong UserDAO.java


public User findByEmail(String email) throws SQLException {
    String sql = "SELECT * FROM Users WHERE email = ?";
    try (Connection conn = DBUtil.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
         
        ps.setString(1, email);
        ResultSet rs = ps.executeQuery();
        if (rs.next()) {
            User user = new User();
            user.setUserId(rs.getInt("user_id"));
            user.setFullName(rs.getString("full_name"));
            user.setEmail(rs.getString("email"));
            user.setPassword(rs.getString("password")); // Hash bảo mật
            user.setRole(rs.getString("role")); // customer
            return user;
        }
    }
    return null; // Không tìm thấy
}

2.2 Bổ sung hàm registerUserByEmail trong UserDAO.java

Mật khẩu của tài khoản tạo bằng Google sẽ được set random (UUID) vì người dùng không nhập pasword. Họ có quyền "Quên mật khẩu" nếu muốn dùng login truyền thống sau này.

public User registerExternalUser(String email, String fullName) throws SQLException {
    String sql = "INSERT INTO Users (full_name, email, password, role) VALUES (?, ?, ?, 'customer')";
    try (Connection conn = DBUtil.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) {
         
        ps.setString(1, fullName);
        ps.setString(2, email);
        
        // Random mật khẩu phức tạp để bảo báo người dùng không tự login được nếu ko dùng oauth (trừ khi dùng chức năng quên pass)
        String randomComplexPass = java.util.UUID.randomUUID().toString();
        ps.setString(3, PasswordUtil.hashPassword(randomComplexPass)); 
        
        int rows = ps.executeUpdate();
        if (rows > 0) {
            ResultSet rs = ps.getGeneratedKeys();
            if (rs.next()) {
                int newId = rs.getInt(1);
                // Return user mới tạo
                User newUser = new User();
                newUser.setUserId(newId);
                newUser.setFullName(fullName);
                newUser.setEmail(email);
                newUser.setRole("customer");
                return newUser;
            }
        }
    }
    return null;
}

3. Service & Xử lý Nghiệp Vụ (Business Logic)

Quá trình logic xử lý khi Google trả về như sau:

  1. Google Web trả JSON (Chứa thư mục Email + Name của Google Account).
  2. Dùng Jackson để parse JSON này (ví dụ lấy: jsonNode.get("email").asText()).
  3. Lấy email gửi cho hàm Service sau: processGoogleLogin(email, name).

Mã nguồn trong UserService.java


public User processGoogleLogin(String email, String fullName) {
    try {
        UserDAO dao = new UserDAO();
        User existingUser = dao.findByEmail(email);
        
        // Cốt lõi: Email làm định danh
        if (existingUser != null) {
            return existingUser; // Nếu có, đăng nhập luôn (Dù trước đây tạo bằng tay hay google)
        } else {
            // Nếu chưa có, tự động tạo mới người dùng
            return dao.registerExternalUser(email, fullName);
        }
    } catch (SQLException e) {
        e.printStackTrace();
        return null;
    }
}

4. Flow Hoạt Động Cụ Thể Tóm Tắt (Direction)

  • Bước 1 (User Request): Người dùng click "Login with Google" -> Browser điều hướng đến Auth Endpoint Google.
  • Bước 2 (Google Callback): Google xác thực xong, gọi ngược về `GoogleCallbackServlet` của dự án qua Method GET kèm mã code.
  • Bước 3 (Backend Request): Servlet của ta gọi POST ngầm ở Server tới Google để đổi code lấy access_token.
  • Bước 4 (Lấy Profile): Dùng Token đó gọi get User Info Google. Chuỗi trả về là JSON. Dùng thư viện Jackson bạn tự cài để extract email.
  • Bước 5 (Database Query): Chạy API processGoogleLogin(email, name) bên trên.
  • Bước 6 (Create Session): Lấy được Đối tượng User, lưu nó vào HttpServletRequest Session, cuối cùng Redirect PRG về trang /home.
