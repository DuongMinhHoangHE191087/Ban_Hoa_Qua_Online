Bản Trình Bày & Hướng Dẫn Code: Đăng ký, Đăng nhập (Local) & Google OAuth

Chào bạn (HoangDM), Đây là tài liệu quy trình chuẩn do tôi xây dựng. Việc Đăng ký/Đăng nhập luôn là Core Logic quan trọng nhất. Xin hãy đọc kỹ luồng xử lý (Code Flow) bên dưới. Có thể copy - paste vào IDE (NetBeans) của bạn và hoàn thiện nó một cách chuẩn chỉnh nhất.

Kiến trúc chuẩn của Dự Án (Luồng Request Xử lý)

Luồng Request: JSP (UI) ➔ Servlet (Controller) ➔ ValidationUtil (Kiểm tra dữ liệu) ➔ AuthService (Xử lý nghiệp vụ) ➔ UserDAO (Lưu CSDL).

Lưu ý: Mọi Request thay đổi dữ liệu (như Đăng nhập, đăng ký) BẮT BUỘC dùng POST và phải Redirect (PRG Pattern) để tránh Double Submit. Xử lý Flash Message báo lỗi qua SessionUtil rồi mới redirect.

PHẦN 1: QUY TRÌNH LOGIN & REGISTER CƠ BẢN (LOCAL ACCOUNT)

1. Cập nhật AuthService.java

Trong AuthService chúng ta xử lý logic băm mật khẩu, kiểm tra trùng email. Sử dụng HashUtil.java (ví dụ hàm băm BCrypt hoặc SHA) vốn đã có của bạn.


// Trong package com.fruitmkt.service;

import com.fruitmkt.dao.UserDAO;
import com.fruitmkt.model.entity.User;
import com.fruitmkt.util.HashUtil;
import java.sql.SQLException;

public class AuthService {
    private final UserDAO userDAO = new UserDAO();

    // 1. Đăng ký tài khoản (Dành cho Customer tự nhập Form)
    public User register(String fullName, String email, String plainPassword, String phone) throws Exception {
        if (userDAO.findByEmail(email) != null) {
            throw new Exception("Email này đã tồn tại trong hệ thống!");
        }

        // Băm mật khẩu để bảo mật trước khi đưa xuống DAO
        // Lưu ý: Nếu HashUtil có hàm hashPassword, hãy dùng nó.
        String hashedPass = HashUtil.hashPassword(plainPassword); 

        // Hàm save hoặc insert của DAO
        int insertedId = userDAO.saveNewCustomer(fullName, email, hashedPass, phone);
        if (insertedId > 0) {
            // Lấy lại user vừa tạo để Set vào session
            return userDAO.findByEmail(email); 
        }
        throw new Exception("Lỗi hệ thống khi tạo tài khoản.");
    }

    // 2. Đăng nhập cơ bản
    public User login(String email, String plainPassword) throws Exception {
        User user = userDAO.findByEmail(email);
        if (user == null) {
            throw new Exception("Tài khoản hoặc mật khẩu không chính xác.");
        }

        if ("INACTIVE".equals(user.getStatus())) {
            throw new Exception("Tài khoản của bạn đã bị vô hiệu hoá.");
        }

        // Kiểm tra đối chiếu hash (Tuỳ vào HashUtil bạn đang viết)
        if (!HashUtil.checkPassword(plainPassword, user.getPasswordHash())) {
            // Có thể thêm logic: userDAO.incrementFailedLogin(user.getUserId());
            throw new Exception("Tài khoản hoặc mật khẩu không chính xác.");
        }

        // Thành công: Xóa biến đếm số lần sai mật khẩu
        // userDAO.resetFailedLogin(user.getUserId());
        return user;
    }
}

2. Cập nhật LoginServlet.java / RegisterServlet.java (Controller)

Servlet chỉ chịu trách nhiệm nhận tham số, gọi Validation và gọi Service. Dưới đây là mẫu chuẩn mực (Code an toàn, không có dính logic query).


@WebServlet("/register")
public class RegisterServlet extends HttpServlet {
    private final AuthService authService = new AuthService();

    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        
        request.setCharacterEncoding("UTF-8");
        
        // 1. Lấy Parameters
        String fullName = request.getParameter("fullName");
        String email = request.getParameter("email");
        String password = request.getParameter("password");
        String phone = request.getParameter("phone");

        // 2. Validation (Xử lý lỗi chuẩn)
        if (!ValidationUtil.isValidEmail(email)) {
            SessionUtil.setFlashMessage(request, "error", "Email không hợp lệ.");
            response.sendRedirect(request.getContextPath() + "/register.jsp"); // Đảm bảo PRG
            return;
        }
        if (password == null || password.length() < 6) {
            SessionUtil.setFlashMessage(request, "error", "Mật khẩu phải từ 6 ký tự.");
            response.sendRedirect(request.getContextPath() + "/register.jsp");
            return;
        }

        // 3. Gọi Tầng Service (Try Catch)
        try {
            User newUser = authService.register(fullName, email, password, phone);
            
            // 4. Set Session & Redirect Success
            SessionUtil.setLoggedUser(request, newUser);
            SessionUtil.setFlashMessage(request, "success", "Đăng ký thành công!");
            response.sendRedirect(request.getContextPath() + "/home");

        } catch (Exception e) {
            // Có lỗi validate từ Logic (Trùng email...)
            SessionUtil.setFlashMessage(request, "error", e.getMessage());
            response.sendRedirect(request.getContextPath() + "/register.jsp");
        }
    }
}

PHẦN 2: QUY TRÌNH GOOGLE OAUTH 2.0 (BẰNG JAVA THUẦN)

Vì sao gọi là chuẩn: Chức năng này lấy tài khoản Google và ánh xạ với `Email` trong UserDAO. Nếu User từng đăng nhập tay bằng `A@gmail.com`, giờ dùng Google Login chung Email đó, hệ thống sẽ gộp Session làm một, tránh sinh rác dữ liệu.

1. Service: Cập nhật hàm processGoogleLogin (Đã setup lần trước)

Hàm này nằm chung trong AuthService.


    // Xử lý logic Oauth Data
    public User processGoogleLogin(String email, String fullName) throws Exception {
        User existingUser = userDAO.findByEmail(email);
        if (existingUser != null) {
            if ("INACTIVE".equals(existingUser.getStatus())) {
                throw new Exception("Tài khoản của bạn đã bị vô hiệu hoá.");
            }
            return existingUser; 
        } else {
            // Sinh mật khẩu random an toàn vì Oauth không cung cấp pass
            String randomPass = java.util.UUID.randomUUID().toString();
            String hashedPass = HashUtil.hashPassword(randomPass);

            // Insert role mặc định CUSTOMER qua DAO
            int newId = userDAO.saveNewCustomer(fullName, email, hashedPass, null);
            return userDAO.findByEmail(email);
        }
    }

3. Servlet Đón Mã Code từ Google (GoogleCallbackServlet.java)

Đây là Servlet phức tạp nhất, dùng Gson/Jackson để parse cục JSON từ Google về Java.


@WebServlet("/GoogleCallback")
public class GoogleCallbackServlet extends HttpServlet {
    private final AuthService authService = new AuthService();

    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        
        String code = request.getParameter("code");
        String error = request.getParameter("error");

        // Nếu user bấm Hủy (Cancel auth)
        if (error != null) {
            SessionUtil.setFlashMessage(request, "error", "Đăng nhập Google bị hủy.");
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }

        try {
            // Bước 1: Fetch Access Token từ Google
            String accessToken = getToken(code);
            
            // Bước 2: Dùng Access Token Get Info User (Tên, Email)
            GoogleAccount acc = getUserInfo(accessToken);

            // Bước 3: Đưa xuống Tầng Service Auth
            User user = authService.processGoogleLogin(acc.getEmail(), acc.getName());

            // Bước 4: Tạo Session và Điều hướng
            SessionUtil.setLoggedUser(request, user);
            SessionUtil.setFlashMessage(request, "success", "Welcome back, " + user.getFullName() + "!");
            response.sendRedirect(request.getContextPath() + "/home");

        } catch (Exception e) {
            e.printStackTrace();
            SessionUtil.setFlashMessage(request, "error", "Lỗi liên kết Google: " + e.getMessage());
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }

    // -- Các hàm Utils fetch HTTP (Dùng core Java http connection) --
    
    private String getToken(String code) throws Exception {
        // ... (Viết luồng tạo POST request tới GoogleConstants.GOOGLE_LINK_GET_TOKEN đính kèm CLIENT_ID, SECRET, CODE)
        // Parse "access_token" JSON bằng JsonUtil/Jackson/Gson và return string
        return "parsed_token_example"; 
    }

    private GoogleAccount getUserInfo(String accessToken) throws Exception {
        // ... (Viết luồng tạo GET request tới GoogleConstants.GOOGLE_LINK_GET_USER_INFO + accessToken)
        // Lấy cục JSON trả về, parse thành class (POJO) chứa Email và name
        return new GoogleAccount("hoang@gmail.com", "Hoàng DM");
    }
}
Tôi đã viết logic getToken và getUserInfo ở dạng Method Signature (khung). Lát nữa khi code, bạn có thể search "Call POST Json Java 11 HttpUrlConnection" kết hợp Jackson / JsonUtil trong util để extract (Bóc tách) node JSON vì dự án hiện bạn tự chọn config thư viện parse Json (Gson/Jackson).
