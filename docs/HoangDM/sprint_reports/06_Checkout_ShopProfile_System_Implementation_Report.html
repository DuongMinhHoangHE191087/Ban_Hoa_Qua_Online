2. Phân Hệ Thanh Toán & Đặt Hàng (Checkout & Order Placement)

Hệ thống hỗ trợ lưu thông tin và thanh toán an toàn cho người dùng thông qua mô hình ACID Transaction kết nối với SQL Server:

A. Thực Thể Địa Chỉ Người Dùng (UserAddress Model)

Bổ sung bảng dữ liệu địa chỉ liên kết với tài khoản người dùng để phục vụ quá trình đặt hàng nhanh chóng.

public class UserAddress {
    private int addressId;
    private int userId;
    private String recipientName;
    private String phoneNumber;
    private String addressLine;
    private boolean isDefault;
}

B. Quy Trình Tạo Đơn Hàng An Toàn (CheckoutServlet)

Quy trình đặt hàng được quản lý dưới dạng JDBC Transaction với cơ chế khóa dòng sản phẩm để tránh Over-selling:

1
Xác thực & Giỏ Hàng: Lấy thông tin giỏ hàng từ cơ sở dữ liệu dựa trên phiên đăng nhập của người dùng.
2
Khởi tạo Transaction: Thiết lập connection.setAutoCommit(false).
3
Trừ Tồn Kho & Tạo Đơn: Cập nhật số lượng sản phẩm, lưu đơn hàng Order, các OrderItem tương ứng và commit đồng thời.

3. Phân Hệ Cửa Hàng & Shop Profile

Hỗ trợ chủ cửa hàng cập nhật ảnh bìa, ảnh đại diện và các thông tin mô tả chi tiết thông qua luồng Servlet API bảo mật:

  • ShopProfileUploadAPI: Tiếp nhận ảnh tải lên dạng Multipart, mã hóa tên tệp và lưu vào thư mục tài sản tĩnh `/assets/images/shops/`.
  • ShopView Page: Trang hiển thị storefront động của riêng từng shop, nạp danh sách nông sản thực tế kèm bộ lọc.

4. Chức Năng Bảo Trì Hệ Thống (Force Logout)

Cho phép Admin phát lệnh cưỡng chế đăng xuất toàn bộ người dùng khỏi hệ thống thông qua việc thu hồi tất cả Token Cookie và làm trống bảng user_sessions để sẵn sàng cập nhật tính năng mới.
