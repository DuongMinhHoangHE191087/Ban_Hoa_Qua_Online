Bảo mật và luồng Auth

Tài liệu ngắn mô tả luồng đăng ký, đăng nhập, và các điểm bảo mật đã áp dụng sau khi loại bỏ debug log nhạy cảm và siết cookie.

HttpOnly Secure khi HTTPS SameSite=Lax PRG sau POST Không log mật khẩu

1. Kết luận về localhost

Chạy http://localhost không bị ảnh hưởng về chức năng. Cookie vẫn được set, chỉ là không có cờ Secure vì kết nối không phải HTTPS.

Nếu chạy qua HTTPS nội bộ hoặc reverse proxy có header X-Forwarded-Proto: https, cookie sẽ tự động được gắn Secure.

2. Luồng đăng ký

[Register form] | | POST /auth/register v [RegisterServlet] | | 1. Check CSRF | 2. Validate input | 3. Hash password | 4. Save user inactive | 5. Send verification email v [AuthService + UserDAO] | v [Redirect /auth/verify]
  • Không còn debug log đọc toàn bộ tham số POST.
  • Mật khẩu được hash trước khi lưu xuống DAO.
  • User được tạo ở trạng thái chưa kích hoạt cho tới khi xác minh email.

3. Luồng đăng nhập

[Login form] | | POST /auth/login v [LoginServlet] | | 1. Check CSRF | 2. AuthService.login() | 3. Invalidate session cũ | 4. Create new session | 5. Issue accessToken + refreshToken v [Role dashboard]
  • Giảm rủi ro session fixation bằng cách hủy session cũ trước khi tạo session mới.
  • Token cookie được gắn HttpOnly, SameSite=Lax, và Secure khi chạy trên HTTPS.
  • Redirect sau login đã chặn đường dẫn ngoài ngữ cảnh ứng dụng.

4. Các điểm kiểm soát bảo mật

Input validation

Kiểm tra email, số điện thoại, mật khẩu, và tên hiển thị trước khi persistence.

Password hashing

Mật khẩu chỉ đi qua HashUtil.hashPassword() trước khi lưu.

CSRF

Luồng POST kiểm tra token CSRF trước khi xử lý nghiệp vụ.

Cookie hardening

Cookie auth không để lộ qua JavaScript và hạn chế gửi trong bối cảnh cross-site.
