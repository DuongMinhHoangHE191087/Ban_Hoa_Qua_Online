C. Sản phẩm: lấy all / filter / API — tư duy + đủ hướng đi

C1. “Lấy toàn bộ sản phẩm” nghĩa là gì trong project này?

Không có API “SELECT * FROM products không điều kiện” cho guest. Public list luôn bị siết bởi:

  • status = 'ACTIVE'
  • approval_status = 'APPROVED'
  • Shop owner profile APPROVED
  • Còn variant active có stock
  • Season window (nếu set)
  • Phân trang (PAGE_SIZE_PRODUCTS)

Cách gọi “all public” bạn đã có:

  • HTML: GET {context}/products
  • JSON trang 1: GET {context}/products?format=json
  • JSON trang n: ?format=json&page=n

Admin/shop list là API/servlet khác (AdminProductServlet, ProductManageServlet) — đừng trộn visibility public với admin.

File 3 — Service: ProductService.java

public PagedResultDTO getProductList(int page, String keyword, Integer categoryId,
        BigDecimal minPrice, BigDecimal maxPrice) throws SQLException {
    return getProductList(page, keyword, categoryId, minPrice, maxPrice, null);
}

public PagedResultDTO getProductList(int page, String keyword, Integer categoryId,
        BigDecimal minPrice, BigDecimal maxPrice, Boolean isOrganic) throws SQLException {
    if (page < 1) page = 1;
    if (minPrice != null && maxPrice != null && minPrice.compareTo(maxPrice) > 0) {
        BigDecimal tmp = minPrice; minPrice = maxPrice; maxPrice = tmp; // hoặc throw
    }
    int pageSize = AppConfig.PAGE_SIZE_PRODUCTS;
    int total = productDAO.countSearch(keyword, categoryId, minPrice, maxPrice, isOrganic);
    int totalPages = Math.max(1, (int) Math.ceil((double) total / pageSize));
    if (page > totalPages) page = totalPages;
    List<Product> items = productDAO.search(
            keyword, categoryId, minPrice, maxPrice, isOrganic, page, pageSize);
    return new PagedResultDTO(items, page, totalPages, total, pageSize);
}

// Servlet

// DAO — map whitelist
private String resolveOrderBy(String sort) {
    if ("price_asc".equals(sort)) return "min_price ASC, p.product_id DESC";
    if ("price_desc".equals(sort)) return "min_price DESC, p.product_id DESC";
    if ("rating".equals(sort)) return "p.rating DESC, p.product_id DESC";
    return "p.product_id DESC"; // default newest
}

@WebServlet("/api/products")
public class ProductApiServlet extends HttpServlet {
    private final ProductService productService = new ProductService();

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        req.setCharacterEncoding("UTF-8");
        try {
            // parse y hệt ProductListServlet (copy helper hoặc extract util parse)
            PagedResultDTO result = productService.getProductList(...);
            Map<String, Object> data = new LinkedHashMap<>();
            data.put("items", result.getItems()); // hoặc map DTO nhẹ hơn Product entity
            data.put("page", result.getCurrentPage());
            data.put("totalPages", result.getTotalPages());
            data.put("totalItems", result.getTotalItems());
            data.put("pageSize", result.getPageSize());
            JsonUtil.writeJson(resp, ApiResponse.ok(data));
        } catch (IllegalArgumentException ex) {
            resp.setStatus(400);
            JsonUtil.writeJson(resp, ApiResponse.fail(ex.getMessage()));
        } catch (Exception ex) {
            resp.setStatus(500);
            JsonUtil.writeJson(resp, ApiResponse.fail("Không tải được sản phẩm."));
        }
    }
}

D. Đơn hàng: thêm input — tư duy + đủ hướng đi

D1. Hiểu “form → DB” bằng 5 cửa (bắt buộc nắm)

CửaFileViệcNếu quên thì sao
1. UI name checkout.jsp name="myField" trong form POST + CSRF Browser không gửi → server null
2. Parse CheckoutServlet#buildCheckoutRequest req.getParameter("myField") → DTO DTO null dù form có gửi
3. DTO CheckoutRequestDTO field + getter/setter không compile / không mang data
4. Business map CheckoutService#validateRequest + buildOrder validate + order.setXxx DB nhận default/null sai rule
5. Persist OrderDAO.save(conn, ...) + schema cột INSERT + mapRow SELECT mất dữ liệu sau redirect; detail không hiện
Hầu hết bug “tôi đã thêm input mà không lưu” là thiếu 1 trong 5 cửa — thường cửa 2 hoặc cửa 5 (save(conn)).

D5. CSRF, PRG, Encoding — không được “tối ưu bỏ”

  • CSRF: form có name="_csrf"; servlet check trước placeOrder. Thiếu → flash lỗi, redirect cart.
  • PRG: sau POST thành công chỉ redirect — không forward JSP kết quả trên POST (tránh F5 mua 2 lần; project còn user lock).
  • UTF-8: req.setCharacterEncoding("UTF-8") đầu doPost/doGet.
  • XSS: in ra JSP bằng <c:out> / fn:escapeXml.
  • Validate server: JS chỉ phụ; rule thật ở validateRequest.
